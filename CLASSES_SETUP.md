# Настройка классов

## Формат файла classes.txt

Создайте файл `classes.txt` в корне проекта (или в `data/` или `config/`) с перечислением классов, по одному на строку:

```
helmet
vest
```

**Важно:**
- Порядок важен: первая строка = класс 0, вторая = класс 1, и т.д.
- Пустые строки игнорируются
- Строки, начинающиеся с `#`, считаются комментариями и игнорируются

## Примеры

### Минимальный пример (2 класса):
```
helmet
vest
```

### С комментариями:
```
# Защитная каска
helmet

# Сигнальный жилет
vest
```

### Больше классов:
```
helmet
vest
gloves
boots
```

## Расположение файла

Файл `classes.txt` ищется в следующем порядке:
1. `classes.txt` (корень проекта)
2. `data/classes.txt`
3. `config/classes.txt`

## Что происходит после создания файла

После создания файла `classes.txt`:
- Классы автоматически загружаются при запуске любого скрипта
- Конфигурация датасета (`config/ppe_data.yaml`) создается автоматически с этими классами
- Детектор использует эти классы для отображения результатов

## Проверка

После создания файла `classes.txt`, запустите:

```bash
python -c "from src.utils.config import config; print('Классы:', config.classes)"
```

Должно вывести загруженные классы.
